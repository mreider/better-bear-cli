better-bear-cli

A CLI and MCP server for Bear notes. Talk to your notes from the terminal or let AI assistants read, search, and edit them via CloudKit.

Download GitHub
curl -sL https://raw.githubusercontent.com/mreider/better-bear-cli/main/install.sh | bash

CLI

Create, search, edit, tag, attach files, and manage TODOs from the terminal. Works without Bear open.

MCP Server

19 tools for Claude Desktop. Let AI read, write, and organize your Bear notes.

CloudKit API

Same REST API Bear Web uses. No SQLite, no x-callback-url. Syncs to all devices.

Open Source

MIT licensed. Universal macOS binary. Swift 5.9+.

Install

Quick install

curl -sL https://raw.githubusercontent.com/mreider/better-bear-cli/main/install.sh | bash

Upgrade

bcli upgrade # download and replace with latest bcli upgrade --check # check without installing

After upgrading, run hash -r or open a new terminal. Restart Claude Desktop if you use the MCP server.

Download manually

Grab the latest from GitHub Releases. Universal binary (arm64 + x86_64), macOS 13+.

curl -L https://github.com/mreider/better-bear-cli/releases/latest/download/bcli-macos-universal.tar.gz -o bcli.tar.gz tar xzf bcli.tar.gz mv bcli ~/.local/bin/bcli rm bcli.tar.gz

Build from source

Requires Swift 5.9+.

git clone https://github.com/mreider/better-bear-cli.git cd better-bear-cli swift build -c release cp .build/release/bcli ~/.local/bin/bcli

Auth

bcli auth

Opens your browser for Apple Sign-In via CloudKit JS. Token saved to ~/.config/bear-cli/auth.json. You can also pass a token directly with bcli auth --token '<token>'.

Commands

Notes

bcli lsList notes
bcli ls --all --tag work --jsonFilter by tag, show all, JSON output
bcli ls --untaggedList notes with no tags
bcli ls --archivedList archived notes
bcli ls --trashedList trashed notes
bcli get <id>View note with metadata
bcli get <id> --rawJust the markdown
bcli get <id> --jsonJSON output (includes parsed front matter)

Search

bcli search "query"Full-text search (title, tags, body)
bcli search "q" --since yesterdayModified after date
bcli search "q" --since last-weekRelative: today, yesterday, last-week, last-month, last-year
bcli search "q" --since 2026-01-01Absolute: YYYY-MM-DD
bcli search "q" --before 2026-06-01Modified before date
bcli search "q" --no-syncSearch without syncing first

Create

bcli create "Title" -b "Body"Create a note
bcli create "Title" -t "t1,t2"Create with tags
bcli create "Title" --stdinPipe content from stdin
bcli create "Title" --fm "status=draft" "project=alpha"Create with YAML front matter

Edit

bcli edit <id> --append "text"Append to end of note
bcli edit <id> --append "text" --after "Heading"Insert after a heading or text
bcli edit <id> --replace-section "Heading" --section-content "new"Replace content under a heading
bcli edit <id> --editorOpen in $EDITOR
bcli edit <id> --stdinReplace entire content from stdin
bcli edit <id> --set-fm "status=done" "reviewed=true"Set front matter fields
bcli edit <id> --remove-fm "draft"Remove front matter fields

Attachments

bcli attach <id> photo.jpgUpload image to iCloud, embed in note
bcli attach <id> doc.pdfAttach any file type
bcli attach <id> img.png --prependInsert after the title
bcli attach <id> img.png --after "Profile"Insert after a heading or text
bcli attach <id> img.png --before "Footer"Insert before a heading or text
bcli attach <id> img.png --at-line 5Insert after line 5
bcli attach name.png --base64 <data>Attach from base64 data (for programmatic use)

Tags

bcli tagsTag tree with counts
bcli tag add <id> "work"Add tag to a note
bcli tag remove <id> "work"Remove tag from a note
bcli tag rename "old" "new"Rename tag across all notes
bcli tag delete "tag"Remove tag from all notes

Archive & Trash

bcli archive <id>Archive a note
bcli archive <id> --undoUnarchive
bcli trash <id>Move to trash

TODOs

bcli todoList notes with incomplete TODOs
bcli todo --limit 10Limit results
bcli todo <id>View TODOs in a note (interactive toggle)
bcli todo <id> --toggle 3Toggle item 3 without prompting
bcli todo --jsonJSON output
bcli todo --no-syncSkip auto-sync

Export

bcli export ./dirExport all notes as markdown
bcli export ./dir --frontmatterInclude YAML metadata (merges with existing front matter)
bcli export ./dir --tag workExport only matching tag

Sync & Config

bcli syncSync notes to local cache
bcli sync --fullForce full re-sync
bcli upgradeSelf-update to latest release
bcli upgrade --checkCheck for updates without installing
bcli mcp installInstall MCP server into Claude Desktop
bcli mcp uninstallRemove MCP server from Claude Desktop

MCP Server (Claude Desktop)

19 tools that let Claude read, search, create, edit, tag, and attach files to your Bear notes.

Setup

cd mcp-server && npm install && npm run build && cd .. bcli mcp install # Restart Claude Desktop

To remove: bcli mcp uninstall

Tools

bear_syncSync notes from iCloud
bear_list_notesList notes with tag filter, include archived/trashed
bear_get_noteFull content by ID (includes parsed front matter)
bear_searchFull-text search with since/before date filters
bear_get_tagsTag hierarchy with counts
bear_create_noteCreate with tags and front matter
bear_edit_noteAppend, replace, section edit, front matter set/remove
bear_attach_fileUpload image/file to iCloud with placement options
bear_archive_noteArchive or unarchive
bear_trash_noteMove to trash
bear_add_tagAdd tag to note
bear_remove_tagRemove tag from note
bear_rename_tagRename across all notes
bear_delete_tagDelete from all notes
bear_find_untaggedList untagged notes
bear_list_todosNotes with incomplete TODOs
bear_get_todosTODO items from a note
bear_toggle_todoToggle TODO completion

Locked/private notes show locked: true in results. Their body content may not be searchable.

How it works

This CLI uses the same CloudKit REST API that Bear Web uses. Notes live in your iCloud private database under iCloud.net.shinyfrog.bear. Authentication uses the iCloud web auth token flow. The CLI does not touch Bear's local SQLite database — it's safe to use with Bear open on any device.
